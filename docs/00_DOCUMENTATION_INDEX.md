# Documentation Index

## Naming Convention

All documentation files in this folder follow a **serial numbering system** with the format:
```
NN_DESCRIPTIVE_NAME.md
```

Where:
- `NN` = Two-digit serial number (01, 02, 03, etc.)
- `DESCRIPTIVE_NAME` = Uppercase snake_case description
- `.md` = Markdown file extension

## Adding New Documentation

When adding new documentation:

1. **Determine the appropriate section** based on content
2. **Assign the next available number** in that section range
3. **Use the naming format**: `NN_DESCRIPTIVE_NAME.md`
4. **Update this index file** with the new document
5. **If creating a new section**, use the next available decade range (20-29, 30-39, etc.)

### Suggested Future Section Ranges:
- **20-29**: Testing & Quality Assurance
- **30-39**: Deployment & DevOps
- **40-49**: Feature Specifications
- **50-59**: API Integrations
- **60-69**: Performance & Optimization
- **70-79**: Security & Compliance
- **80-89**: User Guides
- **90-99**: Troubleshooting & FAQ

## Examples

✅ **Good naming**:
- `20_TESTING_STRATEGY.md`
- `21_UNIT_TEST_GUIDE.md`
- `30_DEPLOYMENT_GUIDE.md`

❌ **Bad naming**:
- `testing.md` (no serial number)
- `TestingStrategy.md` (wrong case)
- `20-testing-strategy.md` (hyphens instead of underscore)
